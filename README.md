# 房地产股票 AI 评级 Agent

基于**量化技术分析 + 同花顺iFinD专业数据 + 腾讯混元2.0大模型 + 实时新闻资讯**的中国房地产股票智能评级系统，覆盖 A股、港股、美股三大市场共 55 只房地产相关股票。支持多数据源自动降级、AI联网搜索最新政策，政策资讯对评级的影响权重高达 35%。

## 功能模块

### 1. 用户认证系统
- JWT Token 认证，支持多终端同时登录，token 有效期 72 小时
- 管理员后台创建用户，最大支持 100 个账户
- 角色权限：管理员（全部功能）/ 普通用户（查看功能）

### 2. AI 智能评级（三引擎融合）
- **综合评分 = 量化技术 × 30% + iFinD基本面 × 30% + AI大模型 × 40%**
- 若基本面不可用则自动降级为量化40%+AI60%；若AI不可用则量化50%+基本面50%
- 每日 09:00、12:00、15:00 三时段自动刷新，覆盖 55 只房地产股票
- **iFinD专业数据**：PE/PB/ROE/EPS/资产负债率/资金流/量比/委比/振幅/换手率/连涨跌天数/多周期涨跌幅
- **基本面5维评分**：核心估值(50分)+资金面(20分)+市场情绪(10分)+盈利能力(10分)+交易活跃度(10分)
- **港股数据增强**：iFinD不返回连涨天数和多周期涨跌幅时，自动由历史行情计算补充
- **换手率合理性校验**：自动过滤异常值（如港股返回的非百分比原始数据）
- **混元联网搜索**：开启 EnableEnhancement，AI 可实时搜索最新政策信息
- 评分精度精确到小数点后 2 位
- 支持按市场、评级筛选，支持历史日期回溯

### 3. 股票详情面板（iFinD数据驱动）
- **核心估值**：PE(TTM)、PB(MRQ)、总市值、ROE、EPS、资产负债率、基本面评分
- **多周期涨跌幅**：5日/10日/20日/60日/120日/年初至今，红绿色直观显示
- **资金流向**：主力净流入（含进度条可视化）、散户净流入、超大单净流入
- **市场微观数据**：量比、换手率、委比、振幅、连涨/跌天数
- **公告与财报**：近90天公告，财报/重点公告自动标记，支持PDF下载；点击个股时预加载公告数据
- **AI专业分析**：腾讯混元2.0大模型分析报告，含评分构成公式
- 价格走势图表、评分趋势、历史评级记录

### 4. 地产行业要闻
- **新闻+公告合并展示**：行业新闻 + iFinD公告统一时间线
- **默认展示5条，点击"查看更多"扩展至10条**
- **30天时效过滤**：自动排除旧闻，按时间倒序排列
- **官方权威数据源**：中国政府网、人民网、新华网 + 东方财富 + 同花顺iFinD
- **AI 智能筛选**：混元大模型评分，仅保留高相关度内容（15秒超时自动降级为关键词排序）
- **启动预热缓存**：服务启动时后台预加载新闻缓存，用户首次访问秒级响应
- 5分钟自动刷新

### 5. 多数据源降级
- **四层数据架构**：iFinD专业数据 → akshare（东方财富）→ 腾讯财经 → 新浪财经
- **iFinD覆盖**：历史行情、实时行情（80+指标）、估值、财务、公告
- **智能数据源排序**：连续失败超过 3 次的数据源自动降低优先级

### 6. 市场点评
- 管理员发布每日行业点评和个股点评
- 支持分类筛选、创建、编辑、删除
- **支持分享到微信朋友圈**（生成公开链接，无需登录即可查看）

### 7. 研究报告
- 管理员上传机构研究报告（PDF/Word/Excel/PPT，最大50MB）
- **PDF在线预览**（RFC 5987中文文件名编码），上传成功动画反馈
- **支持分享到微信朋友圈**（含报告摘要、PDF内嵌预览和下载入口）

## 评级模型

### 综合评分 = 量化技术 × 30% + iFinD基本面 × 30% + AI大模型 × 40%

**量化技术评分**（5个维度加权）：

| 维度 | 权重 | 指标 |
|------|------|------|
| 趋势评分 | 25% | 四级均线排列(MA5/10/20/60)、价格偏离度、MA20斜率、均线黏合度、ADX趋势强度 |
| 动量评分 | 20% | RSI双周期(6+14)、MACD信号+柱状体变化、KDJ随机指标、Williams %R、多周期涨跌幅 |
| 波动率评分 | 15% | 年化波动率、布林带宽度+价格位置、ATR(14)相对波动、波动率收敛/发散趋势 |
| 成交量评分 | 20% | 多级量比(5/10/20日)、OBV能量潮、VWAP偏离度、量价配合度、成交量趋势 |
| 价值评分 | 20% | 区间位置连续评分、筹码集中度、多级支撑压力(10/20/60日)、价格动态区间 |

**iFinD基本面评分**（五维度100分制，动态归一化）：

| 维度 | 满分 | 指标 |
|------|------|------|
| 核心估值 | 50分 | PE(TTM) 15分 + PB(MRQ) 14分 + ROE 11分 + 资产负债率(三道红线) 10分 |
| 资金面 | 20分 | 主力净流入 10分 + 量比 6分 + 委比 4分 |
| 市场情绪 | 10分 | 连涨/跌天数 5分 + 振幅 5分 |
| 盈利能力 | 10分 | EPS每股收益 10分 |
| 交易活跃度 | 10分 | 换手率 5分 + 20日涨跌幅 5分 |

**AI大模型评分**（腾讯混元2.0 Think + 联网搜索 + iFinD数据）：

| 维度 | 权重 | 说明 |
|------|------|------|
| 政策资讯影响 | 35% | 最新房地产政策、调控文件、行业新闻、iFinD公告，实时联网获取 |
| 公司基本面 | 30% | 经营质量、财务健康(三道红线)、土储质量、销售回款、管理层能力 |
| 技术面与资金面 | 25% | 价格趋势、iFinD资金流向(主力/散户)、北向资金、筹码结构分析 |
| 风险评估 | 10% | 债务违约风险、项目交付风险、系统性行业风险 |

**评级映射**：

| 综合评分 | 评级 |
|----------|------|
| ≥ 65 | 优选 |
| 50 - 64 | 关注 |
| 35 - 49 | 中性 |
| < 35 | 谨慎 |

> 若 AI 不可用，自动降级为量化+基本面评级；若基本面也不可用，降级为 100% 量化评分。

## 技术栈

**后端**：
- FastAPI + Uvicorn（异步 Web 框架）
- SQLAlchemy + aiosqlite（异步 ORM + SQLite）
- APScheduler（定时任务，每日 09:00 / 12:00 / 15:00 三时段自动评级）
- **同花顺iFinD HTTP API**（历史行情/实时行情/估值/财务/资金流/公告查询）
- AKShare + 腾讯财经 + 新浪财经（多数据源降级，自动重试）
- httpx（异步调用腾讯混元 API，支持联网搜索）
- pandas / numpy（量化计算）
- python-jose + passlib（JWT 认证 + 密码哈希）
- aiofiles（异步文件上传）

**前端**：
- React 18 + Vite（构建工具）
- Recharts（价格走势和评分趋势图表）

**部署**：
- Docker + Docker Compose
- Nginx 反向代理（限流 + 静态缓存 + PDF代理优化）

## 快速开始

### 1. 环境准备

```bash
git clone <repo-url>
cd "real estate stock agent"
```

### 2. 配置环境变量

```bash
cd backend
cp .env.example .env
```

编辑 `backend/.env`：

```env
HUNYUAN_SECRET_ID=你的SecretId
HUNYUAN_SECRET_KEY=你的SecretKey
HUNYUAN_MODEL=hunyuan-2.0-thinking-20251109

JWT_SECRET=修改为随机字符串
ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin123

# 同花顺iFinD API
IFIND_REFRESH_TOKEN=你的refresh_token
IFIND_ACCESS_TOKEN=你的access_token
```

### 3. Docker 部署（推荐）

```bash
docker compose up -d --build
```

访问 http://localhost，使用管理员账号登录。

### 4. 本地开发运行

**启动后端**：

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

**启动前端**：

```bash
cd frontend
npm install
npm run dev
```

访问 http://localhost:3000（Vite代理后端API至8000端口）

### 5. 默认账号

首次启动自动创建管理员：`admin` / `admin123`（可通过 .env 修改）。

普通用户由管理员在"用户管理"页面创建，最大支持 100 个账户。
